# Milestone 1: Crystal Run single-agent foundation

M1 establishes the scenario, event, fairness, Teacher, and evaluation contracts
that later learned policies must use. It does not claim multiplayer or learned
policy performance.

## Accepted evidence

The official held-out test contains 100 frozen seeds for each capability. All
five Teachers completed 100/100 episodes, the Wilson 95% lower bound is 0.963
for every capability, and the event decoder recorded zero inconsistencies.

| Capability | Teacher | Gate | Result |
|---|---|---:|---:|
| Navigation | `fixed_route` | ≥95% | 100% |
| Pickup | `objective_first` | ≥95% | 100% |
| Return | `evasive_return` | ≥95% | 100% |
| Static hit | `aggressive_script` | ≥90% | 100% |
| Moving hit | `aggressive_script` | ≥75% | 100% |

Machine-readable artifacts are in `reports/m1/`: `episodes.csv` contains all
500 traces, `summary.json` contains the gate decision, and `manifest.json`
binds the result to the frozen split, scenario manifest, and Git commit.

## Prerequisites and deterministic build

Use the Python 3.10 `botcolosseo` Conda environment. The scenario build requires
ACC 1.60 and its include directory; FFmpeg is required for videos.

```bash
conda activate botcolosseo
python -m pip check
python scripts/check_env.py
ACC_PATH=/path/to/acc ACC_INCLUDE=/path/to/acc/source \
  python scripts/build_crystal_run.py --check
```

The build check compiles the tracked UDMF/ACS sources in a temporary directory
and requires byte-identical WAD and manifest outputs.

## Test and smoke commands

```bash
python -m ruff check src tests scripts
python -m pytest tests/unit -v
python -m pytest tests/integration -v
python scripts/smoke_crystal_run.py \
  --task navigation --teacher fixed_route
python scripts/smoke_crystal_run.py \
  --task moving_hit --teacher aggressive_script \
  --record videos/m1-moving-hit.mp4 --require-video
```

The other valid task/Teacher pairs are `pickup/objective_first`,
`return/evasive_return`, and `static_hit/aggressive_script`.

## Reproduce the quantitative gate

```bash
python scripts/evaluate_m1.py --split test --output reports/m1
python scripts/render_m1_showcase.py
```

Official status is available only for the complete 500-case test manifest.
Partial runs require `--development`; their summaries set `official: false` and
cannot pass the gate.

## Failure diagnosis

- `set_seed()` type errors indicate a stale manifest or wrong evaluator; tracked
  M1 seeds are unique signed-32-bit-safe integers.
- An ACC failure should be diagnosed from the compiler output and include path;
  do not edit the compiled WAD directly.
- A task success without a `task_success` event, or the reverse, is a protocol
  inconsistency and fails the official gate regardless of win rate.
- Missing or empty video output requires a working FFmpeg/libx264 installation.
- If `python` resolves outside the project Conda environment, reactivate it or
  use `conda run -n botcolosseo python` explicitly.
