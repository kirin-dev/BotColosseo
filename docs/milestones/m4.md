# M4 Aggressive status

Status: **in progress; two endpoint routes are verified and the Pareto midpoint
selection is awaiting approval**.

## What completed

The first residual-adapter candidate completed 400,000 environment steps,
1,444 episodes, and 6,252 PPO updates with zero KL early stops. Its final
checkpoint SHA-256 is
`426a956d58330b1c0edbb117e3a1f0f167034744f10bcafefbce1f79b5d6c12d`.
Training used validation-free league inputs and reports
`test_cases_accessed: false`.

The implementation includes a frozen fair-observation base actor, a
zero-initialized residual style adapter, a copied style policy head,
`D_KL(style || base)`, capped reward components, style-checkpoint inference,
and an append-only paired evaluation path.

## Honest result

A 20-episode validation smoke compared the frozen style-base candidate and the
400k Aggressive candidate across all five script opponents with side swaps.
The ledger was complete and protocol-clean. Overall Skill Retention was 90%,
but neither policy selected an attack action; engagement initiation and valid
attack rates were therefore zero. The Aggressive candidate failed
`engagement_shift`, `valid_attack_rate`, and the per-opponent retention floor.

The raw append-only evidence and its hash-bound manifest are under
`reports/m4/smoke/aggressive-0400000/`. This smoke is a diagnostic result, not
an M4 pass and not publication metric evidence.

## Root cause and proposed correction

The original 100k BC demonstration set contains no learner attack actions.
With the base actor frozen, valid-hit shaping is too sparse to move greedy
inference across the attack-action exploration barrier. Running the full
200-episode evaluation would only repeat an already explicit failure.

The approved correction was success-filtered Aggressive teacher distillation:
generate public-observation attack opportunities with the existing privileged
teacher, retain only attacks that produce a valid hit, pretrain only the style
adapter and copied policy head, then resume task-reward PPO with KL and the
existing anti-hacking penalties.

## Success-filtered route result

The production data pass generated 50,000 transitions across all five frozen
opponents. It retained 7,857 successful attacks and 3,318 sampled non-attack
contexts while excluding 28,874 unsuccessful attacks from supervision. The
1,000-update adapter pretraining gate passed: attack probability on retained
positive observations increased from 0.00054 to 0.916, while the negative
attack prediction rate remained 0.0645. No test cases were accessed.

The subsequent 400,000-step PPO run completed 1,402 episodes and 6,252 updates
with zero KL early stops. Its learner produced 997 valid hits during sampled
training, compared with 36 for the first reward-only candidate. The 400k
checkpoint SHA-256 is
`632812b1b94412d44d1b86302bb88b0b86c1290f781f39223b39bd30f938e188`.

## Verified Pareto endpoints

Three protocol-clean 20-episode validation smokes expose where deterministic
greedy behavior changes:

| Candidate | Skill retention | Attack rate | Valid attack rate | Result |
| --- | ---: | ---: | ---: | --- |
| distilled adapter | 0.850 | 0.1705 | 0.1117 | visible but over-aggressive |
| PPO 200k | 0.975 | 0.0086 | 0.2000 | capable but too sparse |
| PPO 400k | 1.000 | 0.0000 | 0.0000 | style erased at greedy inference |

The distilled endpoint passes the engagement-shift gate but fails the
objective-chase, valid-attack, and per-opponent-retention gates. The 200k
checkpoint passes global and per-opponent retention plus the valid-attack
threshold, but its engagement confidence interval still includes zero and
three of five attacks chase the objective carrier. The 400k checkpoint fully
retains capability but never selects an attack under deterministic inference.

Evidence is preserved under `reports/m4/smoke/aggressive-distillation/`,
`reports/m4/smoke/aggressive-distilled-0200192/`, and
`reports/m4/smoke/aggressive-distilled-0400000/`. These are diagnostic
validation results, not an M4 pass.

The proposed next step is a validation-smoke grid over fixed checkpoint
interpolations between the distilled style branch and the 200k style branch.
Only the adapter and copied policy head are interpolated; the frozen Strong
Base remains byte-identical. This creates a product-facing style-strength
control while preserving deterministic public-observation inference. It
requires route approval before implementation.
